# worklog:project — 同步项目文档

检测当前项目，分析代码库，更新项目文档。

## 配置

```
VAULT_PATH = "/Users/liutianpeng/projects/worklog"
```

## 流程

当此命令被调用时：

---

### 阶段 1：项目检测

1. **识别当前项目**：
   - 扫描 `VAULT_PATH/项目/` 中所有 `.md` 文件
   - 读取每个文件的 `**Location:**` 字段（如果存在）与 cwd 比较
   - 或将 cwd 目录名与项目文件名模糊匹配
   - 如果未找到匹配，进入"创建模式"

2. **如果找到**，读取现有文件内容作为更新基准。

---

### 阶段 2：代码库分析

从代码库收集项目数据：

1. **技术栈检测** — 读取 `go.mod`、`package.json`、`requirements.txt`、`Cargo.toml` 等文件
2. **目录结构** — 扫描 1-2 层深度
3. **近期 git 历史** — `git log --oneline -10`
4. **Git 远程地址** — `git remote get-url origin`（如果存在）
5. **关键源文件** — 识别入口点、主要模块及其作用

---

### 阶段 3：操作菜单

**如果项目文件已存在：**

| 选项 | 说明 |
|------|------|
| 更新描述与技术栈 | 刷新项目描述和技术栈信息 |
| 添加开发记录条目 | 在项目文档中添加今日记录 |
| 更新关键文件表格 | 用当前代码库刷新关键文件部分 |
| 创建/更新架构画布 | 生成 `.canvas` 架构图 |
| 完整同步（上述全部） | 执行所有更新 |

**如果项目文件不存在：**

| 选项 | 说明 |
|------|------|
| 创建新项目文件 | 在 `项目/` 创建 `{项目名}.md` |
| 创建项目文件 + 架构画布 | 同时创建文档和画布 |

---

### 阶段 4：执行选定操作

#### 操作：创建新项目文件

构建文件：
```markdown
# 项目名称

## 简介
{2-3 句话说明项目功能，根据代码库分析编写}

**Location:** `{绝对路径}`
**GitHub:** {远程 URL 如果可用}

---

## 技术栈
- {技术1} — {用途}
- {技术2} — {用途}

---

## 关键文件

| 文件 | 作用 |
|------|------|
| `{路径}` | {功能} |

---

## 开发记录

### YYYY-MM-DD
- {完成的工作}
```

保存到 `VAULT_PATH/项目/{项目名}.md`

---

#### 操作：添加开发记录条目

在项目文档的 `## 开发记录` 部分添加今日条目（新条目在最前）：
```markdown
### YYYY-MM-DD
- {从 git 历史 和会话上下文综合的工作描述}
```

---

#### 操作：创建架构画布

1. 询问用户详细程度：
   - 高层系统（15-30 个节点）
   - 文件级细节（40+ 个节点）

2. 分析代码库识别架构层级

3. 构建 `.canvas` JSON 文件，保存到 `VAULT_PATH/画布/{项目名}架构.canvas`

---

### 阶段 5：报告

```
## 同步完成

**项目：** {名称}
**文件：** {路径}

**执行的操作：**
- [x] {操作1}
- [x] {操作2}
```
