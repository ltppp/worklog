# worklog:summarize — 整理本周笔记到资源

从日记中提取笔记，整理到资源文件夹。

## 配置

```
VAULT_PATH = "/Users/liutianpeng/projects/worklog"
```

## 用法

```
/worklog:summarize           # 默认整理本周
/worklog:summarize last-week # 整理上周
```

## 流程

当此命令被调用时：

---

### 1. 确定时间范围

- `本周`：从本周一到今天
- `last-week`：上周一到上周日

计算日期范围。

---

### 2. 扫描日记文件

读取 `VAULT_PATH/日记/` 中符合时间范围的所有 `.md` 文件。

---

### 3. 提取笔记候选

从每个日记的 `## 笔记` 部分提取内容。

**过滤已整理：** 跳过以下标记的条目：
- `→ [[资源名]]` — 已整理到指定资源
- `(已整理)` — 已整理但未指定资源

示例：
```markdown
## 笔记
- MongoDB 连接池默认最大连接数是 100 → [[MongoDB]]  ← 已整理，跳过
- Go context 传递技巧                                          ← 未整理，候选
- gRPC 调试方法：用 grpcurl 测试                               ← 未整理，候选
```

---

### 4. 展示候选列表，交互选择

```
本周笔记候选（N 条未整理）：

1. [✓] MongoDB 连接池默认最大连接数是 100
2. [ ] Go context 传递技巧
3. [✓] gRPC 调试方法：用 grpcurl 测试
...

选择要整理到资源的条目：
- 默认全选（直接回车）
- 输入编号取消选择（如输入 "2" 取消第 2 条）
- 输入多个编号（如 "2 4" 取消多条）
```

---

### 5. 整理到资源

根据笔记内容主题，决定：

- **合并到现有资源**：如 MongoDB 相关笔记追加到 `资源/MongoDB.md`
- **创建新资源**：如 `资源/gRPC.md`

追加格式：
```markdown
## 资源名

### 子主题
- {笔记内容}
- {另一条相关笔记}
```

---

### 6. 标记已整理

回到日记文件，给已整理的条目加标记：

```markdown
## 笔记
- MongoDB 连接池默认最大连接数是 100 → [[MongoDB]]
- Go context 传递技巧 → [[Go技巧]]
- gRPC 调试方法：用 grpcurl 测试 → [[gRPC]]
```

---

### 7. 报告结果

```
整理完成：
- 更新 资源/MongoDB.md（2 条）
- 创建 资源/gRPC.md（1 条）
- 标记 3 条已整理
```

---

## 标记规则

| 标记 | 含义 |
|------|------|
| `→ [[资源名]]` | 已整理到该资源文件，点击可跳转 |
| `(已整理)` | 已整理但未指定具体资源文件 |
| 无标记 | 未整理，下次执行时仍会出现在候选列表 |

---

## 注意

- 可以重复执行，已标记的条目不会重复整理
- 用户可以选择部分整理，未选的保留在日记中
- 资源文件名由 Claude 根据笔记内容智能推断
