# worklog:summarize — 整理本周笔记到资源

从日记中提取笔记，整理到资源文件夹。

## 配置

```
VAULT_PATH = "/Users/liutianpeng/projects/worklog"
```

## 用法

```
/worklog:summarize           # 整理本周
/worklog:summarize last-week # 整理上周
```

## 流程

---

### 1. 确定时间范围

- 本周：本周一到今天
- last-week：上周一到上周日

---

### 2. 扫描日记文件

读取 `VAULT_PATH/日记/` 中符合时间范围的 `.md` 文件。

---

### 3. 提取笔记候选

从 `## 笔记` 部分提取，**跳过已标记的**：
- `→ [[资源名]]` — 已整理
- `(已整理)` — 已整理

---

### 4. 交互选择

```
本周笔记候选（N 条未整理）：

1. [✓] MongoDB 连接池默认最大连接数是 100
2. [ ] Go context 传递技巧
...

选择要整理的条目（默认全选，输入编号取消）
```

---

### 5. 整理到资源

- 合并到现有资源文件，或创建新文件
- 按主题智能分类

---

### 6. 标记已整理

```markdown
- MongoDB 连接池默认最大连接数是 100 → [[MongoDB]]
```

---

### 7. 报告结果

```
整理完成：
- 更新 资源/MongoDB.md（2 条）
- 创建 资源/gRPC.md（1 条）
- 标记 3 条已整理
```

## 标记规则

| 标记 | 含义 |
|------|------|
| `→ [[资源名]]` | 已整理到该资源，可点击跳转 |
| `(已整理)` | 已整理但未指定资源 |
| 无标记 | 未整理，下次仍会出现 |
