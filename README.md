# worklog

极简的个人工作管理知识库。

**只需记住一件事：打开今天的文件，写。**

无需编号系统、无需插件、无需复杂分类。每天一个文件，所有内容先记这里，重要的再整理。

## 安装

```bash
git clone https://github.com/你的用户名/worklog.git
```

1. 用 Obsidian 打开 `worklog` 文件夹作为知识库
2. 复制命令文件到 Claude 配置目录：
   ```bash
   cp CLAUDE.md ~/.claude/commands/obsidian.md
   ```
3. 编辑命令文件，修改 `VAULT_PATH` 为你的知识库绝对路径：
   ```bash
   VAULT_PATH = "/Users/你的用户名/projects/worklog"
   ```

## 结构

```
日记/     ← 每天一个文件，所有内容先记这里
项目/     ← 项目文档，技术细节、决策记录
资源/     ← 可复用的知识点、参考文档
画布/     ← Canvas 文件，可视化架构图
```

## 工作流

1. **上班** → 打开今天的日记，写任务、记进展
2. **下班** → 把重要的技术细节整理到项目文档
3. **周末** → 把可复用的知识点整理到资源

## `/obsidian` 命令

从任意项目目录运行 `/obsidian`，Claude 会：

- 将当前目录匹配到知识库中的项目文件
- 分析代码库（技术栈、目录结构、git 历史）
- 提供操作菜单：
  - 创建/更新项目文档
  - 添加今日日志
  - 生成架构画布

### 配置位置

命令文件位于 `CLAUDE.md`，需要复制到 `~/.claude/commands/obsidian.md` 并修改 `VAULT_PATH`。

## 文件命名

- 日记：`YYYY-MM-DD.md`（如 `2026-04-03.md`）
- 项目：`{项目名}.md`（如 `Octopus.md`）
- 画布：`{项目名}架构.canvas`

## 日记模板

```markdown
# YYYY-MM-DD

## 今天要做的
- [ ] 任务列表

## 工作记录
- 上午：...
- 下午：...

## 笔记
（临时想法、学到的东西）

## 明天继续
- ...
```

## 项目文档模板

```markdown
# 项目名

## 简介
一句话描述。

## 技术栈
- 技术1
- 技术2

## 关键文件
- `路径` - 作用

## 开发记录
### YYYY-MM-DD
- 完成了什么
```
