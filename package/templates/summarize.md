# worklog:summarize — 整理本周笔记到资源

从日记中提取笔记，整理到资源文件夹，建立双向链接。

## 配置

```
VAULT_PATH = "{{VAULT_PATH}}"
```

## 用法

```
/worklog:summarize           # 整理本周
/worklog:summarize last-week # 整理上周
```

## 流程

---

### 1. 确定时间范围

- 本周：本周一到今天
- last-week：上周一到上周日

---

### 2. 扫描日记文件

读取 `VAULT_PATH/日记/` 中符合时间范围的 `.md` 文件。

---

### 3. 提取笔记候选

从 `## 笔记` 部分提取，**跳过已链接的**：
- `→ [[资源名]]` — 已整理
- `(已整理)` — 已整理

---

### 4. 分析主题并归类

AI 分析每条笔记，归类到合适的资源：
- 技术知识点 → `资源/{技术名}.md`
- 最佳实践 → `资源/{主题}最佳实践.md`
- 问题解决方案 → `资源/{主题}.md`

---

### 5. 交互确认

```
本周笔记（N 条未整理）：

1. MongoDB 连接池默认最大连接数是 100
   → 建议归类到 [[MongoDB]]

2. Go context 传递要用 context.WithValue
   → 建议归类到 [[Go并发模式]]

确认整理？(Y/n)
```

---

### 6. 整理到资源

**创建或更新资源文档：**

```markdown
# MongoDB

数据库相关知识点。

## 要点
- 连接池默认最大连接数是 100
- （其他要点...）

## 参考
- [官方文档](https://www.mongodb.com/docs/)
```

**在日记中标记链接：**
```markdown
## 笔记
- MongoDB 连接池默认最大连接数是 100 → [[MongoDB]]
```

---

### 7. 报告结果

```
整理完成：
- 更新 资源/MongoDB.md（2 条）
- 创建 资源/Go并发模式.md（1 条）
- 标记 3 条已链接
```

---

## 资源文档模板

```markdown
# {主题}

一句话描述。

## 要点
- 核心知识点 1
- 核心知识点 2

## 代码示例
\`\`\`
// 可选
\`\`\`

## 参考
- [链接标题](url)
```

---

## 链接规则

| 标记 | 含义 |
|------|------|
| `→ [[资源名]]` | 已整理到该资源，可点击跳转 |
| 无标记 | 未整理，下次仍会出现 |

---

## 设计理念

- 笔记先记在日记，周末整理到资源
- 建立双向链接：日记 → 资源
- 资源文档保持原子化，一个主题一个文件
- Obsidian 自动在资源底部显示反向链接（相关日记）
